# AIエージェント支援ガイド

このドキュメントは、algorithm-golangプロジェクトにおいて、AIエージェントが効果的に開発支援を行うためのガイドラインです。

## 🎯 プロジェクト概要

このリポジトリは、Golangを使用して基本的なソートアルゴリズムを学習・実装するための教育目的のプロジェクトです。

### 主要な目標
- Golangでのアルゴリズム実装の学習
- パフォーマンス計測と計算量の理解
- クリーンで読みやすいコードの実践

## 🤖 AIエージェントの役割

### 1. コード実装支援

#### 新しいアルゴリズムの追加
- `src/`ディレクトリ内に新しいアルゴリズムのフォルダを作成
- `main.go`にアルゴリズムの実装を記述
- 既存のユーティリティ（`pkg/executeTime.go`、`pkg/generateRandIntArray.go`）を活用

#### コーディング規約
- Go標準のコーディングスタイルに従う（`gofmt`準拠）
- 関数とロジックに適切なコメントを記載
- パッケージインポートは標準ライブラリ → サードパーティ → ローカルの順

### 2. ドキュメント更新

#### README.mdの更新
新しいアルゴリズムを実装した際は、以下を更新：
- プロジェクト構造にディレクトリを追加
- 実装済みアルゴリズムのチェックリストを更新（`[ ]` → `[x]`）
- 実行方法に新しいコマンド例を追加

#### コード内ドキュメント
- アルゴリズムの時間計算量・空間計算量を記載
- アルゴリズムの動作原理を簡潔に説明
- 使用例やエッジケースについて言及

### 3. テストとデバッグ

#### 実装後の確認事項
- [ ] コードが正常にコンパイルされるか（`go build`）
- [ ] プログラムが期待通りに動作するか（`go run main.go`）
- [ ] エラーハンドリングが適切か
- [ ] エッジケース（空配列、単一要素など）に対応しているか

#### パフォーマンス検証
- 実行時間を計測（`pkg.Measure`または`pkg.MeasureWithWait`を使用）
- 異なるデータサイズでの動作確認
- 理論的な計算量と実測値の比較
- 必要に応じて`pkg.MeasureConcurrent`で複数アルゴリズムを並行比較

### 4. コードレビューのポイント

#### 確認すべき項目
- **正確性**: アルゴリズムが正しく実装されているか
- **効率性**: 不必要な処理やループがないか
- **可読性**: コードが理解しやすいか
- **保守性**: 将来の変更や拡張が容易か
- **一貫性**: プロジェクト内の他のコードとスタイルが統一されているか

## 📋 タスク別支援ガイドライン

### 新しいソートアルゴリズムの実装

```bash
# 1. ディレクトリ作成
mkdir src/[algorithm-name]

# 2. main.goの作成（テンプレート）
# - パッケージ宣言
# - 必要なインポート
# - アルゴリズム実装関数
# - main関数でのテスト実行

# 3. 動作確認
cd src/[algorithm-name]
go run main.go

# 4. ドキュメント更新
# - README.mdのチェックリスト更新
# - 実行方法の追加
```

### ユーティリティ関数の追加

```bash
# 1. pkg/ディレクトリに新しい.goファイルを作成
# 2. package pkgを宣言
# 3. エクスポート可能な関数を実装（大文字で開始）
# 4. 使用例をREADME.mdに追加
```

### バグ修正

1. 問題の特定と再現
2. 原因の分析
3. 修正の実装
4. テストによる検証
5. 関連ドキュメントの更新（必要に応じて）

## 🔄 開発フロー

1. **要件理解**: ユーザーのリクエストを正確に把握
2. **設計**: 実装方針を決定（既存コードとの整合性を考慮）
3. **実装**: コードを記述
4. **検証**: 動作確認とテスト
5. **ドキュメント**: README.mdやコメントの更新
6. **レビュー**: コード品質の最終確認

## 📝 コミュニケーション

### ユーザーへの報告

実装完了時には以下を報告：
- 実装した内容の概要
- 作成・変更したファイルのリスト
- 実行方法
- 注意点や今後の改善点（あれば）

### 質問が必要な場合

以下の場合はユーザーに確認：
- 仕様が不明確な場合
- 複数の実装方法があり選択が必要な場合
- 既存コードに大きな変更が必要な場合
- プロジェクト構造の変更が必要な場合

## 🎓 アルゴリズム実装の参考情報

### ソートアルゴリズムの基本構造

```go
package main

import (
    "fmt"
    "github.com/jugeeem/algorithm-golang/src/pkg"
)

// [AlgorithmName]Sort は[説明]を行う
// 時間計算量: O(...)
// 空間計算量: O(...)
func [AlgorithmName]Sort(numbers []int) []int {
    // 実装
    return numbers
}

func execute() {
    numbers := pkg.GenerateRandIntArray()
    sortedNumbers := [AlgorithmName]Sort(numbers)
    fmt.Println("Sorted Numbers:", sortedNumbers)
}

func main() {
    // 同期実行の場合
    pkg.Measure("[Algorithm Name]", execute)
    
    // 非同期実行の場合（推奨）
    // pkg.MeasureWithWait("[Algorithm Name]", execute)
}
```

### 計算量の表記

| アルゴリズム | 最良 | 平均 | 最悪 | 空間 |
|-------------|------|------|------|------|
| バブルソート | O(n) | O(n²) | O(n²) | O(1) |
| カクテルソート | O(n) | O(n²) | O(n²) | O(1) |
| 選択ソート | O(n²) | O(n²) | O(n²) | O(1) |
| 挿入ソート | O(n) | O(n²) | O(n²) | O(1) |
| マージソート | O(n log n) | O(n log n) | O(n log n) | O(n) |
| クイックソート | O(n log n) | O(n log n) | O(n²) | O(log n) |

## 🚀 ベストプラクティス

1. **シンプルさを保つ**: 教育目的のプロジェクトなので、理解しやすさを優先
2. **段階的な実装**: 基本的なバージョンから始め、最適化は後回し
3. **豊富なコメント**: アルゴリズムの各ステップを説明
4. **実行例の提供**: 実際の動作結果を示す
5. **一貫性の維持**: 既存コードのスタイルに合わせる
